finding_dismissedNew
Fired when a developer dismisses a finding as a false positive or nitpick — either via a @dryrunsecurity fp / @dryrunsecurity nit GitHub comment from the PR, or via Triage in the Risk Register. Includes the reason, feedback text, who dismissed it, and from where.
{
  "event": "finding_dismissed",
  "event_id": "evt_a1b2c3d0",
  "org": "DryRunSec",
  "repo_name": "dryrun-api",
  "timestamp": "2026-04-10T14:35:00Z",
  "finding_id": "drs_b8948228",
  "finding_type": "Missing Tenant Isolation in Database Delete",
  "severity": "high",
  "filename": "pkg/models/drift.go",
  "dismissal_type": "false_positive", // "false_positive" | "nitpick" | "accepted_risk" | "wont_fix"
  "dismissed_by": "cnichols-dry",
  "dismissed_from": "github_comment", // "github_comment" | "risk_register"
  "feedback": "This code is not user-facing — input is validated upstream by the service layer",
  "pr_number": 412,
  "pr_title": "fix: move db creds to vault",
  "github_comment_url": "https://github.com/DryRunSec/dryrun-api/pull/412#issuecomment-1234567",
  "fingerprint_created": true, // true when dismissal_type is "false_positive" — suppresses in future scans
  "dashboard_url": "https://app.dryrun.security/pull-requests/412?org=DryRunSec"
}
pr_improvedNew
Fired when a PR that had findings on an earlier scan is later scanned again and one or more of those findings are no longer present — indicating the developer fixed issues before merge. All findings must be resolved for all_resolved to be true.

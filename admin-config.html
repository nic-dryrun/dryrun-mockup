GitHub Comment Behavior
Issue Comment Enabled
Master control for all GitHub PR comments. When disabled, DryRun scans PRs but never posts any comment.
Comment on Clean Scans Renamed ? Previously "Show Comment for No Findings." Controls whether DryRun posts a comment when a scan produces zero vulnerabilities.
Post a confirmation comment even when no security vulnerabilities are detected, proving the PR was scanned.
Comment on Draft PRs New ? When enabled, DryRun posts comments on draft PRs just like regular PRs. When disabled, scans still run but comments are held until the PR leaves draft status.
Post findings and scan results on pull requests while they are in draft mode. When disabled, scans run silently and findings appear on the next scan after the PR is marked ready.
Retain Comment When Resolved New ? When enabled, the comment updates to an "All Clear" state showing what was found and how it was handled. When disabled, the comment is deleted once all findings are resolved or dismissed.
Keep the comment with a resolved status and collapsed finding history instead of deleting it when all findings are addressed. Preserves the audit trail.

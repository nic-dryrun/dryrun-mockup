Prompt Injection in Nitpick Filter
Active finding — first detected 3 weeks ago. Present in 4 consecutive scans across this PR. New
Repo DryRunSec/reposture
PR [codex] Add pinecone triage feedback to reviews
#369 by peterkarman1

Description

The _ask_model method in NitpickFilterService constructs a prompt for an LLM by directly concatenating user-controlled finding data (finding.type, finding.description, finding.filename, finding.label) and retrieved data from the Pinecone vector database (doc.page_content in precedents). While the prompt includes a warning to treat data as untrusted, this relies solely on LLM instruction following.

Evidence

The prompt uses direct string concatenation with untrusted input fields from TriageFilterFindingInput (type, filename, label, description).

Precedent content (doc.page_content) retrieved from the database is also concatenated without escaping or clear isolation.
